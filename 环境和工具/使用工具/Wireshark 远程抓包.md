# Windows Wireshark 实时远程抓包 Linux 流量

```
用法：rpcapd [-b <地址>] [-p <端口>] [-t <数据端口>] [-4] [-l <主机列表>] 
            [-a <主机,端口>] [-n] [-v] [-d] [-i] [-D] 
            [-s <配置文件>] [-S] [-C] [-K <ssl密钥文件>] [-X <ssl证书文件>] 
            [-f <配置文件>]

选项：
-b <地址>        绑定到的地址（可以是数字或文本形式）。 
                 默认：绑定到所有本地IPv4和IPv6地址

-p <端口>        绑定的端口。 
                 默认：绑定到端口2002

-t <数据端口>    用于传输数据的端口。
                 默认：由操作系统选择一个未使用的端口

-4               仅使用IPv4。
                 默认：同时使用IPv4和IPv6等待套接字

-l <主机列表>    包含允许连接到此服务器的主机列表的文件（如果多个，请逐行列出）。 
				建议在列表中使用文本名称（而非数字形式），以避免不同地址族的问题。

-n               允许空认证（通常与'-l'一起使用）

-a <主机,端口>   主动模式下连接到'主机'的'端口'（如果省略'端口'，则使用默认端口2003）

-v               仅运行在主动模式（默认：如果指定'-a'，也会接受被动连接）

-d               在守护模式下运行（UNIX系统）或作为服务运行（Windows系统） 
                 警告（Windows）：作为服务从控制面板启动时，此开关会自动提供

-i               在inetd模式下运行（UNIX系统）

-D               记录调试信息

-S               使用SSL加密所有通信（实现rpcaps://协议）

-C               启用压缩

-K <ssl密钥文件> 使用此文件中的SSL私钥（默认：key.pem）

-X <ssl证书文件> 使用此文件中的证书（默认：cert.pem）

-s <配置文件>    将当前配置保存到文件

-f <配置文件>    从文件加载当前配置；命令行中指定的所有开关都将被忽略

-h               打印帮助信息
```
